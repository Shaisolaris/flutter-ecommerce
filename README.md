# flutter-ecommerce

![CI](https://github.com/Shaisolaris/flutter-ecommerce/actions/workflows/ci.yml/badge.svg)

Flutter e-commerce app with product catalog, cart management, wishlist, search with filters and sorting, product detail with variant selection, category browsing, and checkout flow. Dart-only implementation with ChangeNotifier state management and Material Design 3.

## Stack

- **Framework:** Flutter 3.x, Dart 3.3+
- **State:** ChangeNotifier (CartProvider, WishlistProvider)
- **Navigation:** Navigator 1.0 with push/pop
- **Design:** Material Design 3 with custom theme

## Features

### Product Catalog
- Home screen with promotional banner, category chips, sale items carousel, and bestseller grid
- 12 products across 6 categories with realistic data
- Sale badges with discount percentages
- Star ratings and review counts

### Product Detail
- Full-screen product view with description, price, sale indicator
- Variant selectors (size, color, switch type) using ChoiceChips
- Quantity picker with +/- controls
- Add to Cart with variant tracking
- Wishlist toggle with heart icon
- Dynamic price calculation based on quantity

### Cart
- Item list with quantity controls and swipe-to-delete
- Per-item variant display
- Order summary: subtotal, shipping (free over $50), tax
- Free shipping threshold progress indicator
- Empty state with browse CTA
- Clear cart action

### Search & Filtering
- Real-time text search across name, description, and category
- Category filter chips
- Sort options: relevance, price ascending/descending, highest rated
- Result count display
- Bottom sheet sort picker

### Wishlist
- Toggle wishlist from product cards and detail screen
- Heart icon state persistence across screens
- Stored as Set<String> of product IDs

## Key Design Decisions

**ChangeNotifier over BLoC for cart/wishlist.** Cart and wishlist are simple state containers (list of items, set of IDs) that don't benefit from BLoC's stream-based complexity. ChangeNotifier provides direct, synchronous updates with minimal boilerplate.

**Filter engine in the data service.** `MockDataService.filterProducts()` accepts a `ProductFilter` object and returns filtered, sorted results. The search screen builds filters declaratively and passes them to the service. In production, this becomes an API call with query parameters.

**Variant tracking in cart items.** Each CartItem stores `selectedVariants` as a `Map<String, String>`. The same product with different variants (e.g., red M vs blue L) creates separate cart entries. Duplicate detection compares both product ID and variant selections.

**Dart-only, no native code.** Following the portfolio convention, this repo contains only Dart files. Platform directories are generated by `flutter create .` at setup time. This keeps the repo focused on application logic.
